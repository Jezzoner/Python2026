# Python2026 — Ejercicios y ejemplos en Python

**Descripción**
- **Propósito:** Repositorio personal para practicar y aprender Python mediante scripts, ejercicios y pequeñas aplicaciones organizadas por temas.
- **Audiencia:** Estudiantes y desarrolladores que están empezando o quieren repasar fundamentos de Python.

**Estructura**
- **cadenas/**: ejercicios y utilidades para manipulación de cadenas (generadores de email, IDs, etc.).
- **ciclos/**: ejemplos de bucles `for` y `while`, contadores y juegos simples.
- **clases/**: ejemplos de programación orientada a objetos, herencia, encapsulamiento y sistemas pequeños.
- **colecciones/**: listas, diccionarios, sets y ejercicios de estructuras de datos.
- **entradas/**: scripts que leen entradas del usuario y gestionan pequeños flujos.
- **funciones/**: ejemplos de funciones, recursión y organización de código en módulos.
- **operadores/**: ejercicios con operadores, validaciones y lógica básica.
- **poo/**: ejemplos adicionales y subproyectos relacionados con POO.
- **retos/** y **sentencias/**: ejercicios y retos para practicar lógica y estructuras de control.

**Cómo usar**
- **Requisitos:** Python 3.8+ (recomendado 3.10 o superior).
- **Ejecutar un script:** desde la raíz del repositorio ejecuta, por ejemplo:

```
python3 cadenas/generador_email.py
```

- **Editar y experimentar:** abre los archivos con tu editor favorito (PyCharm, VS Code) y modifica los scripts para aprender cómo cambian los resultados.

**Buenas prácticas para estudiar**
- **Lee el código primero:** ejecuta el script y luego revisa el código para entender cada parte.
- **Añade comentarios:** documenta lo que estás probando y por qué.
- **Crea ramas:** si haces cambios grandes, crea una rama Git para mantener el historial limpio.

**Siguientes pasos recomendados**
- Ejecuta los scripts de cada carpeta, comienza por `cadenas/` y `funciones/`.
- Crea ejercicios propios y súbelos como nuevas carpetas o archivos.

**Contacto y contribuciones**
- Este repositorio es para uso personal; si quieres colaborar, abre un issue o un pull request con una descripción clara del ejercicio.
